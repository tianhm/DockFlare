# バックアップと復元

DockFlare 3.0 では完全バックアップアーカイブを利用できます。生データディレクトリに直接触れずに、Master を新しいハードウェアへ移行したり、障害復旧したり、アップグレードの準備を進めたりできます。

## 保存される内容
- `dockflare.key` – すべての暗号化されたファイルのロックを解除する Fernet キー。
- `dockflare_config.dat` – 暗号化された Cloudflare 認証情報、管理画面アカウント、ランタイム設定。
- `agent_keys.dat` – 暗号化されたエージェント API キーと監査メタデータ。
- `state.json` – ルール、エージェント、およびアクセス グループのプレーンな JSON ミラー。
- `manifest.json` – アーカイブのチェックサムとバージョン情報 (自動生成)。

これらのファイルはすべて、単一の `dockflare_backup_YYYYMMDD_HHMMSS.zip` にバンドルされています。 ZIP と抽出したファイルを一緒に保管してください。 `dockflare.key` がないと、暗号化されたアーティファクトは役に立ちません。

## バックアップの作成
1. Master の管理画面で **設定 → バックアップと復元** を開きます。
2. [**バックアップ (.zip) をダウンロード**] をクリックします。
3. アーカイブを安全な場所に保存します。これを認証情報のように扱います。DockFlare を通じて Cloudflare アカウントを制御するために必要なすべてが含まれています。

Master の実行中にバックアップを作成できます。各アーカイブには SHA-256 ハッシュを含むマニフェストが含まれているため、破損したダウンロードを簡単に見つけることができます。

## 既存の Master での復元
1. **[設定] → [バックアップと復元]** に移動します。
2. **バックアップから復元** を使って `.zip` をアップロードします。
3. 警告を確認します: 復元すると構成、エージェント キー、およびルールが上書きされます。

DockFlare は暗号化ファイルを再構成し、`state.json` を再読み込みし、必要に応じて再起動フラグを書き込みます。コンテナは数秒後に終了するため、Docker が新しい構成でコンテナを再起動します。復元された認証情報で管理画面に再びアクセスできるようになります。

従来の `state.json` ファイルは、引き続き部分復元に利用できます。`state.json` 単体をアップロードした場合はルールのみが復元され、暗号化された設定は復元されません（必要なら後から認証情報を再入力してください）。

## セットアップ ウィザード中の復元
新規インストールでは、Pre-Flight ウィザードの Step 1 の前に **バックアップから復元** リンクが表示されます。

1. バックアップ ZIP をアップロードします。
2. DockFlare は、暗号化されたアーティファクトと状態をディスクに書き込みます。
3. コンテナが自動的に再起動します。復帰後、復元された管理者アカウントでサインインします。

このフローは、本番 Master のクローンを作成するか、データ ボリュームをワイプした後に復元する最も速い方法です。ウィザードを再実行したり、Cloudflare 認証情報を再入力したりする必要はありません。

## 復元後
- **[設定] → [バックアップと復元]** にアクセスして、最新のマニフェストのタイムスタンプを確認します。
- [**エージェント → 概要**] をチェックして、登録済みのエージェントが再接続していることを確認します。エージェント キーをローテーションした場合は再発行します。
- 別の環境 (`Actions → Reconcile Now`) に復元した場合は、調整をトリガーします。

定期的にオフライン バックアップを保持し、構成スタックのバージョン管理と組み合わせて、デプロイメント全体を迅速に再構築できるようにします。
