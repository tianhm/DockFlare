# 使用网页管理界面

DockFlare 的网页管理界面用于管理、监控和配置服务。对于超出简单 Docker 标签范围的操作，它提供了更直观的处理方式。

## 仪表板（主页）

登录后看到的第一个页面是主仪表板。这是您查看所有托管服务状态的中央枢纽。

* **托管入口规则表：** 此表列出了 DockFlare 正在管理的每个入口规则，无论它来自 Docker 容器还是手动创建。
    * **主机名：** 服务的公共主机名。
    * **服务：** 内部目标 URL。
    * **来源：** 指示规则来自 `Docker` 还是在界面中手动创建。
    * **状态：** 显示规则是否为 `active`、`pending_deletion` 或已被界面覆盖。
    * **访问：** 显示应用的访问组和模式标记。同步可复用策略后，您会看到“公开”或“需身份验证”等标签、组名以及指向 Cloudflare 仪表板的快捷链接。
    * **管理规则：** 此按钮允许您编辑任何规则。
* **实时日志：** 在表格下方，您将找到一个实时日志查看器，它从 DockFlare 后端流式传输日志，这对于调试非常有用。

## 管理规则

用户界面使您可以完全控制您的入口规则。

* **添加手动规则：** “添加手动规则”按钮可让您为未在 Docker 中运行的服务（例如局域网中另一台计算机上的服务）创建入口规则。该表单允许您指定主机名、服务 URL，并可选择应用访问组。
* **编辑任意规则：** 每个规则旁边的“管理规则”按钮会打开一个弹窗，您可以在其中更改配置。您也可以借此将界面覆盖应用到原本来自 Docker 标签的规则上。
* **恢复为标签配置：** 如果某条 Docker 规则已被界面覆盖，则会出现“恢复到标签”按钮，允许您放弃手动更改并让该规则重新由 Docker 标签控制。

## 访问策略页面

此页面是管理可重复使用的**访问组**并使用通配符策略保护 DNS 区域的中心位置。

### 高级访问策略

从访问组部分，您可以：
* **使用两个选项卡模式（经过身份验证与公共）创建**新的访问组。每个选项卡都会更新指导横幅，以便您了解 DockFlare 何时会发出 Cloudflare `allow` 或 `bypass` 决策。
* **编辑**现有访问组。该模式强制执行特定于模式的验证（经过身份验证所需的电子邮件）并使 Geo/IP 设置对两种模式都可见。
* **删除**不再使用的访问组（无法删除 `public-default-bypass` 等系统策略）。
* **从 Cloudflare 同步**，从您的帐户导入现有的 DockFlare 可重用策略。
* 使用每个条目旁边的操作菜单，通过 Cloudflare 图标快捷方式直接在 Cloudflare 仪表板中打开匹配的策略。

**注意：** `public-default-bypass` 系统策略由 DockFlare 自动创建和管理。使用“绕过”访问的所有服务都会引用此单一策略，从而保持您的 Cloudflare 仪表板干净。

### 区域默认策略（*.tld 通配符）

第二部分显示**区域默认策略** - 保护所有子域的安全最佳实践功能：

* **保护状态：** 视觉徽章显示哪些 DNS 区域具有通配符 `*.domain.com` 策略（受保护 🛡️），哪些没有（不受保护 ⚠️）。
* **创建区域策略：** 在任何未受保护的区域上单击“创建策略”以创建通配符访问应用程序。
* **选择策略：** 选择哪个访问组应保护该区域的所有子域（可以是公共旁路、身份验证或任何自定义策略）。
* **安全安全网：** 即使您忘记向特定服务添加策略，区域级通配符策略也会捕获它。

**最佳实践：** 为所有域创建区域默认策略。对于面向公众的域，请使用默认绕过策略。对于内部/私有域，使用身份验证策略。这可确保不会意外暴露子域。

有关更多详细信息，请参阅[访问策略最佳实践和示例](Access-Policy-Best-Practices.md) 指南。

## 设置页面

设置页面包含各种管理和配置选项：

* **Cloudflare Tunnels：** 此部分列出了您账户中的所有 Cloudflare Tunnels、其状态以及所连接的 `cloudflared` 代理。您还可以查看指向任意 Tunnel 的 CNAME DNS 记录。
* **备份和恢复：** 下载包含加密配置、代理密钥和状态的完整 DockFlare 备份存档 (`.zip`)，或上传之前导出的存档以恢复实例。
* **安全：**
    * **更改密码：** 更改网页管理界面的密码。
    * **禁用密码登录：** 仅适用于将 DockFlare 放在外部身份验证代理之后的高级场景。**⚠️ 警告：** 由于 Docker 网络暴露，这会带来安全风险：同一 Docker 网络上的任意容器都可能绕过外部认证并直接访问 DockFlare API。我们强烈建议使用 OAuth/OIDC 提供程序，以便在不牺牲安全性的前提下获得 SSO 的便利。完整安全影响请参阅 [访问网页管理界面](Accessing-the-Web-UI.md#禁用密码登录)。
* **Cloudflare 凭证：** 允许您在初始设置后更新您的 Cloudflare 帐户 ID 和 API 令牌。
* **核心配置：** 允许您更改隧道名称和规则宽限期等设置。
