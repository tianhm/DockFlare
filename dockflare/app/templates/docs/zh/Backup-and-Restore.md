# 备份与恢复

DockFlare 3.0 引入了完整备份归档，因此您可以将主控端迁移到新硬件、在发生故障后恢复，或在不直接操作原始数据目录的情况下准备升级。

## 保存什么
- `dockflare.key` – 用于解密每个受保护文件的 Fernet 密钥。
- `dockflare_config.dat` – 加密保存的 Cloudflare 凭据、界面账户和运行时设置。
- `agent_keys.dat` – 加密保存的 Agent API 密钥和审计元数据。
- `state.json` – 规则、代理端和访问组的明文 JSON 镜像。
- `manifest.json` – 归档的校验和与版本信息，由系统自动生成。

所有这些文件都会被打包到同一个 `dockflare_backup_YYYYMMDD_HHMMSS.zip` 中。请将 ZIP 文件和解压出的文件一起保存；如果没有 `dockflare.key`，加密归档将无法使用。

## 创建备份
1. 在主界面中打开 **设置 → 备份和恢复**。
2. 点击 **下载备份 (.zip)**。
3. 将归档保存在安全的位置。请把它当作敏感凭据处理，因为其中包含通过 DockFlare 管理 Cloudflare 账户所需的全部内容。

备份可以在主控端运行时创建。每个归档都包含带有 SHA-256 哈希值的 manifest，因此很容易识别损坏或不完整的下载。

## 恢复到现有主控端
1. 进入**设置→备份和恢复**。
2. 通过 **从备份恢复** 上传 `.zip` 文件。
3. 确认警告：恢复会覆盖当前配置、Agent 密钥和规则。

DockFlare 会重新写入加密文件、重新加载 `state.json`，并在需要时设置重启标志。几秒后容器会自动退出，以便 Docker 使用新配置将其重新启动。恢复完成后，网页管理界面会使用已恢复的凭据重新提供服务。

对于部分恢复，仍然支持旧版 `state.json` 文件。上传单独的 JSON 文件只会替换规则，不会覆盖加密配置。

## 在安装向导中恢复
现在，全新安装会在初始设置向导的第 1 步之前显示 **从备份恢复** 链接。

1. 上传备份 ZIP。
2. DockFlare 会将加密文件和状态写入磁盘。
3. 容器会自动重启；重新上线后，请使用恢复的管理员账户登录。

这是克隆生产主控端或在清空数据卷后执行恢复的最快方式。您无需重新运行向导，也不必再次输入 Cloudflare 凭据。

## 恢复后
- 访问 **设置 → 备份和恢复**，确认 manifest 中最新的时间戳。
- 检查 **代理 → 概述**，确认已注册的代理端是否重新连接。如果您轮换过 Agent 密钥，请重新签发。
- 如果您是恢复到不同环境中，请触发一次协调（`Actions → Reconcile Now`）。

请定期执行离线备份，并尽量将其与 Compose 栈的版本控制配合使用，这样在需要时可以更快重建整个部署。
